# Cinemap Launch Checklist

This checklist is the operating version of the plan: ship the trust layer, measure real usage, then spend the 5,000 SAR on attention instead of internal build work.

## 1. Product Must Be Ready

- [ ] Homepage uses the Arabic brand line: `سينماب · أفلام عليها كلام`.
- [ ] Ticket 3 is QA'd on row and modal: personal rating badge, score copy, first-rating prompt.
- [ ] Rating sheet requires consent before local save + Formspree send.
- [ ] Notify sheet requires consent before Formspree send.
- [ ] Add-to-calendar sheet works from row and modal.
- [ ] Footer links include About, Privacy, Terms, and TMDB attribution.
- [ ] `hello@cinemap.me` is active and monitored.

## 2. Legal And Trust

- [ ] Privacy page is live.
- [ ] Terms page is live.
- [ ] About page is live.
- [ ] TMDB attribution appears in footer and privacy/about pages.
- [ ] Data deletion request process is written down: user emails `hello@cinemap.me`, you search Formspree submissions by email, then delete or anonymize.
- [ ] Do one legal review pass before paid distribution. The current pages are founder-friendly baseline copy, not legal advice.

## 3. Tracking And Weekly Review

- [ ] Local event capture is live for saves, reminders, ratings, shares, trailer clicks, calendar adds, and watchlist clears.
- [ ] Formspree submissions are exported weekly.
- [ ] Create one simple Google Sheet with weekly rows: visits, saves, reminders, ratings, shares, top movies, top cities.
- [ ] Review every Sunday for 15 minutes. No feature debate before checking the numbers.

## 4. First 30 Testers

- [ ] DM 30 trusted Saudi movie watchers.
- [ ] Ask each person to save 3 movies and rate 1 watched movie.
- [ ] Watch for confusion around: rating, notify consent, calendar add, and share copy.
- [ ] Fix only critical clarity bugs before public push.

## 5. Founder Distribution

- [ ] Draft launch post: "أفلام عليها كلام".
- [ ] Draft founder story post: why Cinemap exists for Saudi movie fans.
- [ ] Record a 20-30 second screen video showing save, watched, rating, and score.
- [ ] Post in 5 Saudi cinema/lifestyle communities.
- [ ] DM 20 micro-creators with a personal note and the video.

## 6. 5,000 SAR Cash Budget

- Distribution tests and boosts: 3,000 SAR.
- Arabic editorial/movie takes: 800 SAR.
- Social/share design assets: 500 SAR.
- Privacy/terms/PDPL template review: 400 SAR.
- Tooling buffer: 300 SAR.

## 7. Kill Criteria

- After 30 days: if there are fewer than 1,000 visits/week, distribution is the bottleneck.
- After 60 days: if there are fewer than 50 ratings, the rating layer needs a clearer reason to exist.
- After 90 days: if saves are strong but ratings are weak, add one short explanation to the rating sheet and test for two more weeks before cutting scope.
- If no channel drives repeat visitors, pause new features and focus only on distribution tests.

## 8. Do Not Build Yet

- User accounts.
- Native app.
- Booking/ticketing.
- Paid consumer subscription.
- Full Saudi movie database ops.
- Recommendation engine.
- Friend planning.

## 9. Next Build Order

1. Finish tracking and QA.
2. Ship My 2026.
3. Ship share-as-image from My 2026.
4. Run first 1,000-user distribution push.
5. Prepare the first free monthly demand report sample.
